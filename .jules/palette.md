## $(date +%Y-%m-%d) - Prevent screen readers from missing disabled buttons
**Learning:** Native `disabled="true"` attributes completely remove buttons from the tab order. This makes it impossible for screen reader users to discover them or read their explanatory `title` tooltips, breaking the UX when they need to know *why* an action isn't available.
**Action:** For core actions like "Execute Build", replace native `disabled` with `aria-disabled="true"`, ensuring to add a `title` tooltip and implementing a JavaScript click guard to block functionality manually while preserving keyboard access and styles.
