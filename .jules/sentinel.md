## 2024-05-31 - [CRITICAL] Command Injection via GitHub Actions Context Interpolation
**Vulnerability:** User inputs (`${{ inputs.* }}`) were being directly interpolated into bash scripts within `run:` blocks, allowing for arbitrary command execution if an attacker provides a crafted input payload. Furthermore, dynamic arguments were executed using an unsafe `eval` statement.
**Learning:** Even internal tooling or pipeline scripts must treat inputs as untrusted. GitHub Actions context interpolation into a bash script is executed by replacing the template with the string literal before bash parses it, breaking out of standard escaping.
**Prevention:** Always assign GitHub context variables to the step's `env:` block and reference the environment variable within the bash script (e.g., `$MY_VAR`). Instead of dynamically building a string and using `eval` for command arguments, construct a bash array (e.g., `MY_ARRAY+=("-x" "$val")`) and expand it securely (e.g., `"${MY_ARRAY[@]}"`).
