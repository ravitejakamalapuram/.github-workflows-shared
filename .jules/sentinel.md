## 2025-02-28 - [CRITICAL] Command Injection via GitHub Actions Inputs in eval
**Vulnerability:** A script within a composite action (`composite-actions/chrome-extension/package/action.yml`) accepted an input via `${{ inputs.exclude-patterns }}` and passed it directly to a script string running inside an `eval "zip -r ... $EXCLUDE_ARGS"` command. This allowed direct command injection from untrusted inputs in the GitHub Action runner.
**Learning:** In bash, using string interpolation for inputs within inline scripts that execute `eval` or similar string-evaluation functions poses a severe command injection vulnerability. Action inputs aren't automatically sanitized against shell injection.
**Prevention:** Avoid `eval` entirely. Use bash arrays (e.g., `ARGS=()`, `ARGS+=("$INPUT")`) to dynamically construct arguments for shell commands. Additionally, always bind GitHub Action `inputs` to environment variables (`env: VAR_NAME: ${{ inputs.input-name }}`) and reference the environment variable inside the `run:` block instead of interpolating inputs inline using `${{ inputs... }}`.
